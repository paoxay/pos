## 🗄️ ໂຄງສ້າງຖານຂໍ້ມູນ

### 1. `employees` - ພະນັກງານ/ຜູ້ໃຊ້ລະບົບ
```sql
| Column     | Type                        | Description           |
|------------|-----------------------------|-----------------------|
| id         | INT (PK, AUTO_INCREMENT)    | ID ພະນັກງານ           |
| name       | VARCHAR(100)                | ຊື່-ນາມສະກຸນ          |
| username   | VARCHAR(50) UNIQUE          | ຊື່ຜູ້ໃຊ້              |
| password   | VARCHAR(255)                | ລະຫັດຜ່ານ (Plain Text!) |
| role       | ENUM('admin','employee')    | ບົດບາດ               |
| status     | ENUM('active','inactive')   | ສະຖານະ               |
| created_at | TIMESTAMP                   | ວັນທີສ້າງ             |
```

### 2. `products` - ສິນຄ້າ
```sql
| Column          | Type                      | Description                |
|-----------------|---------------------------|----------------------------|
| id              | INT (PK, AUTO_INCREMENT)  | ID ສິນຄ້າ                   |
| barcode         | VARCHAR(50) UNIQUE        | ລະຫັດບາໂຄດ                  |
| name            | VARCHAR(200)              | ຊື່ສິນຄ້າ                   |
| stock           | INT                       | ຈຳນວນໃນສະຕັອກ                |
| cost            | DECIMAL(10,2)             | ຕົ້ນທຶນ                     |
| price           | DECIMAL(10,2)             | ລາຄາຂາຍ                    |
| barcode_printed | TINYINT(1)                | ສະຖານະພິມບາໂຄດ (0/1)        |
| created_at      | TIMESTAMP                 | ວັນທີເພີ່ມ                  |
| updated_at      | TIMESTAMP                 | ວັນທີແກ້ໄຂລ່າສຸດ            |
```

### 3. `sales` - ບັນທຶກການຂາຍ
```sql
| Column      | Type                     | Description               |
|-------------|--------------------------|---------------------------|
| id          | INT (PK, AUTO_INCREMENT) | ID ບິນຂາຍ                  |

| total       | DECIMAL(10,2)            | ຍອດລວມສຸດທ້າຍ              |

| sale_date   | TIMESTAMP                | ວັນທີຂາຍ                  |
```

### 4. `sale_items` - ລາຍການສິນຄ້າໃນບິນ

## 🔐 ລະບົບຄວາມປອດໄພ (Security)

### ✅ ສິ່ງທີ່ມີຢູ່ແລ້ວ

| Feature | ລາຍລະອຽດ | ຕຳແໜ່ງໃນ Code |
|---------|---------|--------------|
| **Session-based Auth** | ກວດສອບ `$_SESSION['user_id']` | `config.php` → `checkLogin()` |
| **Role-based Access** | Admin vs Employee | `config.php` → `checkAdmin()` |
| **ປິດບັງຂໍ້ມູນລັບ** | ພະນັກງານເບິ່ງກຳໄລບໍ່ໄດ້ | `get_sales.php`, `get_sale_detail.php` |
| **ປ້ອງກັນລຶບ Admin ຫຼັກ** | `id != 1` | `employees.php` |
| **Transaction Safety** | ໃຊ້ `beginTransaction()` + `rollback()` | `process_sale.php`, `delete_sale.php`, `import_json.php` |
| **Barcode Unique** | ບໍ່ມີລະຫັດສິນຄ້າຊ້ຳ | Database UNIQUE constraint |

### ⚠️ ຈຸດອ່ອນຄວາມປອດໄພ (ຕ້ອງແກ້ໄຂ)

| ບັນຫາ | ລະດັບຄວາມສ່ຽງ | ວິທີແກ້ໄຂ |
|-------|-------------|----------|
| **ລະຫັດຜ່ານ Plain Text** | 🔴 ສູງຫຼາຍ | ໃຊ້ `password_hash()` + `password_verify()` |
| **ບໍ່ມີ CSRF Token** | 🟠 ປານກາງ | ເພີ່ມ CSRF token ໃນ form |
| **API ບາງອັນບໍ່ກວດ Auth** | 🟡 ຕ່ຳ | `search_products.php` ເປີດແບບ public |
| **SQL Prepared Statement** | ✅ ດີແລ້ວ | ໃຊ້ PDO prepared statements |

## 🌐 API Endpoints

### 1. ການຂາຍ
| Method | Endpoint | ລາຍລະອຽດ | Auth |
|--------|----------|---------|------|
| POST | `process_sale.php` | ບັນທຶກການຂາຍໃໝ່ | ✅ Required |
| GET | `get_sales.php?start_date=&end_date=` | ດຶງປະຫວັດການຂາຍ | ✅ Required |
| GET | `get_sale_detail.php?id=` | ດຶງລາຍລະອຽດບິນ | ✅ Required |
| POST (JSON) | `delete_sale.php` | ລຶບບິນ + ຄືນ Stock | ✅ Required |

### 2. ສິນຄ້າ
| Method | Endpoint | ລາຍລະອຽດ | Auth |
|--------|----------|---------|------|
| GET | `search_products.php?q=` | ຄົ້ນຫາສິນຄ້າ | ❌ Public |
| GET | `add_product_print.php?action=get_recent` | ສິນຄ້າລ່າສຸດ | ✅ Required |
| GET | `add_product_print.php?action=get_dates` | ວັນທີທີ່ເພີ່ມສິນຄ້າ | ✅ Required |
| POST | `add_product_print.php` (action=mark_printed) | ອັບເດດສະຖານະພິມ | ✅ Required |

### 3. Request/Response Format

**ການຂາຍ (process_sale.php)**
```json
// Request (POST - JSON Body)
{
  "items": [
    {"id": 12, "quantity": 1, "price": 250, "cost": 120}
  ],
  "subtotal": 250,
  "discount": 0,
  "total": 250,
  "profit": 130
}

// Response
{
  "success": true,
  "sale": {
    "id": 50,
    "date": "2026-01-05 14:30:00",
    "items": [...],
    "total": 250
  }
}
```

## 🛠️ ສິ່ງທີ່ຕ້ອງລະວັງເວລາພັດທະນາ

### 1. Database
- ✅ ໃຊ້ PDO prepared statements ສະເໝີ
- ✅ ໃຊ້ Transaction ສຳລັບການດຳເນີນງານຫຼາຍຂັ້ນຕອນ
- ⚠️ ລະວັງ CASCADE DELETE ໃນ `sale_items`

### 2. Session
- Session ເລີ່ມໃນ `config.php`
- ຕົວແປ Session ສຳຄັນ:
  - `$_SESSION['user_id']` - ID ຜູ້ໃຊ້
  - `$_SESSION['user_name']` - ຊື່ຜູ້ໃຊ້
  - `$_SESSION['user_role']` - ບົດບາດ (admin/employee)

### 3. Barcode
- ລະບົບປ່ຽນ barcode ເປັນ **UPPERCASE** ອັດຕະໂນມັດ
- barcode ຕ້ອງ Unique ໃນລະບົບ

### 4. Stock Management
- ຕັດ Stock ອັດຕະໂນມັດເມື່ອຂາຍ
- ຄືນ Stock ເມື່ອລຶບບິນ

### 5. Currency
- ອັດຕາແລກປ່ຽນ: 1 THB = X LAK
- ຄ່າປັດຈຸບັນ: 1 THB ≈ 690 LAK

## 💡 Flow ການເຮັດວຽກຫຼັກ

### Flow ການຂາຍ
```
1. ພະນັກງານ Login → index.php
2. ໄປທີ່ແຖບ "ຂາຍສິນຄ້າ"
3. ຄົ້ນຫາ/ສະແກນບາໂຄດ → search_products.php
4. ເລືອກສິນຄ້າ + ຈຳນວນ
5. ໃສ່ສ່ວນຫຼຸດ (ຖ້າມີ)
6. ກົດ "ບັນທຶກການຂາຍ" → process_sale.php
7. ລະບົບ: ບັນທຶກ sales + sale_items + ຕັດ stock
8. ພິມໃບບິນ (Print Dialog)
```

### Flow ການເພີ່ມສິນຄ້າ + ພິມບາໂຄດ
```
1. ໄປທີ່ add_product_print.php
2. ປ້ອນຂໍ້ມູນສິນຄ້າ (barcode/ຊື່/ຕົ້ນທຶນ/ລາຄາ)
3. ກົດ "ບັນທຶກ" → ເພີ່ມລົງ Queue ພິມ
4. ເລືອກສິນຄ້າຈາກລາຍການລ່າສຸດ (ຖ້າຕ້ອງການ)
5. ກົດ "ພິມບາໂຄດ" → ເປີດໜ້າພິມ 40mm x 20mm
6. ລະບົບ: ອັບເດດ barcode_printed = 1
```
